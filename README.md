# CRM для рекламного агентства — backend

## Стек
Node.js + Express + PostgreSQL (Sequelize ORM) + Telegram-бот на Claude API

## Установка

```bash
npm install
cp .env.example .env
# заполните .env: доступы к PostgreSQL, JWT_SECRET, ANTHROPIC_API_KEY, TELEGRAM_BOT_TOKEN
```

## Создание таблиц в БД

Убедитесь, что PostgreSQL запущен и в `.env` указана существующая база данных, затем:

```bash
npm run db:sync
```

## Запуск

```bash
npm run dev   # с автоперезапуском (nodemon)
# или
npm start
```

Сервер поднимется на порту из `.env` (по умолчанию 4000).
Если в `.env` указан `TELEGRAM_BOT_TOKEN` — Telegram-бот запустится автоматически вместе с сервером.

## Структура

```
src/
  config/       — подключение к БД, синхронизация таблиц
  models/       — модели Sequelize (User, Client, Deal, Project, Task, Invoice, Lead, ...)
  middleware/   — проверка JWT и ролей
  routes/       — REST API (auth, clients, deals, projects, invoices, leads)
  bot/          — Telegram-бот и клиент Claude API
```

## Роли и доступы

- **admin** (руководитель) — полный доступ ко всем данным
- **manager / employee** (сотрудники) — доступ к своим клиентам, сделкам, проектам
- **client** (личный кабинет) — доступ только к своим проектам и счетам (read-only)

## Как работает бот

1. Клиент пишет в Telegram
2. Бот подтягивает базу знаний агентства (`knowledge_base`) и историю переписки
3. Запрос уходит в Claude API, ответ приходит клиенту
4. Если Claude определяет, что вопрос выходит за рамки FAQ (например, клиент хочет оставить заявку) —
   создаётся запись в `leads`, диалог помечается как переданный человеку, и дальше с клиентом
   должен связаться менеджер вручную (или через отдельное уведомление, которое можно добавить позже)

## Уведомления о сроках проектов

Клиент не имеет отдельного личного кабинета — вместо этого он привязывает Telegram-чат к своей карточке:

1. Менеджер в админке нажимает «Получить ссылку» на карточке клиента (`POST /api/clients/:id/telegram-link`)
2. Клиент переходит по ссылке `t.me/<bot>?start=<token>` и нажимает Start
3. Бот находит клиента по токену и сохраняет `telegram_chat_id` в `clients`
4. Дальше клиент автоматически получает:
   - сообщение сразу при назначении/изменении срока проекта (`endDate`)
   - напоминания за 3 дня и за 1 день до срока (ежедневная проверка в 10:00, `src/bot/notifications.js`)

Токен одноразовый и обнуляется сразу после привязки.
